# /plan [기능 설명]

복잡한 기능을 분석하고 구현 계획을 수립합니다.

## 사용법
```
/plan 동행 매칭 시스템 구현
/plan 실시간 채팅 기능 설계
/plan 알림 시스템 구축
```

## 언제 사용하나요?

### ✅ 사용해야 할 때
- 여러 모듈에 걸친 기능
- 새로운 도메인/모듈 추가
- 복잡한 비즈니스 로직
- 설계 결정이 필요한 경우

### ❌ 사용 안 해도 될 때
- 단순 CRUD
- 한 모듈 내 기능
- 명확한 구현 방법이 있는 경우

## 출력 내용

1. **요구사항 요약**
2. **API 설계** (엔드포인트, 메서드)
3. **데이터 모델** (Entity, DTO)
4. **모듈 간 통신** (QueryService vs Event)
5. **구현 순서** (작업 분해)
6. **고려사항** (보안, 성능)
7. **질문 사항** (불명확한 부분)
